Développement d'un composant d'une solution décisionnelle

Projet


Objectif :
L'objectif de notre projet était de mettre en place une API à partir d'un jeu de données volumineux. Nous avons conçu un logiciel utilisant Docker Desktop pour gérer quatre conteneurs distincts : un pour l'API, un pour la base de données MySQL, un pour le serveur, et un pour l'interface utilisateur.

Méthodologie :
Nous avons commencé par analyser le jeu de données et définir les besoins fonctionnels de l'API. Ensuite, nous avons développé l'API en Python, utilisé MySQL pour le stockage des données, et Docker Desktop pour orchestrer les conteneurs. Enfin, nous avons créé une interface utilisateur avec React JS pour faciliter l'interaction des utilisateurs avec l'API.

Difficultés :
Nous avons rencontré des défis liés à la gestion de la volumétrie des données et à l'intégration des différents composants du système. Configurer Docker Desktop pour faire fonctionner tous les conteneurs ensemble a également été complexe. De plus, l'apprentissage des différentes technologies utilisées a nécessité un temps d'adaptation supplémentaire.

Compétences acquises :
Ce projet m'a permis d'acquérir des compétences en développement d'API avec Python, en gestion de bases de données avec MySQL, et en utilisation de Docker Desktop pour la gestion de conteneurs.
