Conception et implémentation d'une BDR

Projet



La tâche à accomplir dans le cadre de ce projet est la conception et l'implémentation d'une base de données pour la gestion des alertes au sein du Service Départemental d'Incendie et de Secours des Deux-Sèvres (SDIS 79). Cette base de données permettra de gérer les pompiers professionnels et volontaires, les sinistres et les engins.

Pour ce faire, le premier travail à réaliser est la conception du Modèle Conceptuel de Données (MCD) à l'aide de l'outil looping_mcd. Cette étape consiste à définir les tables, les champs et les associations nécessaires pour répondre aux besoins fonctionnels et aux contraintes énoncées dans le cahier des charges. Il est important de penser aux valeurs de chacun des champs et de s'assurer que le MCD est cohérent et complet.

Une fois le MCD conçu, il convient de générer le Modèle Logique de Données (MLD) et le script SQL de création des tables à l'aide de looping_mcd. Le MLD permet de représenter graphiquement la structure de la base de données, tandis que le script SQL permet de créer les tables dans le Système de Gestion de Base de Données (SGBD) choisi, dans ce cas MySQL. Il est important de vérifier les clés étrangères et de s'assurer que le moteur de stockage des tables est bien INNODB.

Après la création des tables, il est nécessaire de les alimenter en insérant au moins 3 tuples dans chaque table à l'aide de requêtes SQL d'insertion. Ces requêtes doivent être écrites dans un script unique SQL et doivent être fonctionnelles. Il est important de s'assurer que les données insérées sont cohérentes et respectent les contraintes énoncées dans le cahier des charges.

Ensuite, il convient d'importer dans MySQL les autres données qui apparaissent au sein des fichiers csv disponibles sur Updago. Cette étape peut être réalisée de deux manières : en créant des feuilles de calcul Excel correspondant aux colonnes des tables et en important directement dans MySQL (bien que cette méthode soit pénalisée), ou en créant un script d'automatisation Python qui fonctionnera pour les prochaines années. Dans tous les cas, il est important de gérer les erreurs et de s'assurer que les données importées sont cohérentes et respectent les contraintes énoncées dans le cahier des charges.

Dans l'ensemble, ce projet nécessite une bonne compréhension des besoins fonctionnels et des contraintes énoncées dans le cahier des charges, ainsi que des compétences en conception et implémentation de base de données, en langage SQL et en Python. Il est important de s'assurer que chaque étape est réalisée de manière cohérente et complète, et de tester régulièrement le travail accompli pour s'assurer qu'il est fonctionnel et respecte les contraintes énoncées dans le cahier des charges.
